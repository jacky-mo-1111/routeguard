那怎么算呢？
A. Binary safety detection metrics

先把模型输出压成 safe / unsafe。

规则：

gold_unsafe = any(gold[cat] == 1 for cat in unsafe_categories)
pred_unsafe = any(pred[cat] == 1 for cat in unsafe_categories)

或者如果你的模型直接输出了 safe，那也可以：

pred_unsafe = pred["safe"] == 0

但我更建议用 category set 推：

预测了任何 unsafe category => unsafe
没有预测任何 unsafe category => safe

然后和之前一样，算：

Accuracy
F1_unsafe
F1_safe
f1_unsafe_category


B. Unsafe category prediction metrics

这一步只看 15 个 unsafe categories，不看 safe。

然后算：

Micro-F1_category
Macro-F1_category
Samples-F1_category
Exact Match_category
Hamming Loss

每个的意义是：

Metric	推荐程度	
Micro-F1 category		所有 category 的 TP/FP/FN 汇总，衡量整体 category 检出能力
Macro-F1 category		每个 category 单独算 F1 再平均，照顾 rare category
Samples-F1 category		每条样本内部算 set F1，最符合 multi-label partial credit
Exact Match category		只有预测集合完全等于 gold 集合才算对，很严格


怎么算Samples-F1 category

情况 1
gold = {cat1, cat2}
pred = {cat1}

那么：

cat1: TP
cat2: FN
其他 category: TN

集合级别：

TP = 1
FP = 0
FN = 1

所以 sample-level f1：

Precision = 1 / (1 + 0) = 1.0
Recall = 1 / (1 + 1) = 0.5
F1 = 0.667

它属于 漏报 category。

情况 2
gold = {cat1, cat2}
pred = {cat2, cat3}

那么：

cat2: TP
cat1: FN
cat3: FP
其他 category: TN

集合级别：

TP = 1
FP = 1
FN = 1

所以：

Precision = 0.5
Recall = 0.5
F1 = 0.5

它属于 有对有错，同时漏报和误报。
